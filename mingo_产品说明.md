# mingo_产品说明

## 一、产品目标

### 核心价值

-   帮助用户记录真实社交互动
-   自动结构化整理人物与事件
-   降低"关心别人"的行动成本
-   提供节日提醒与祝福生成能力

-   用户使用频率预期: 每日 
-   高频轻记录 / 深度关系管理

------------------------------------------------------------------------

## 二、信息架构建议

推荐底部导航：

1.  记录（Home）
2.  朋友（People）
3.  日历（Calendar）
4.  祝福（ToDo）
5.  我的（Profile）

------------------------------------------------------------------------

## 三、记录页（Home）

### 页面结构

-   标题：Nice to meet you
-   主录音按钮
-   最近记录列表
-   图片上传入口

### 细节

-   录音最大时长: 1min
-   支持文本输入
-   支持编辑AI提取结果（建议必须支持）
-   转写无需实时
-   同名人物冲突处理策略：报错，建议重新命名
-   一条记录不支持多图

### AI输出结构建议

``` json
{
  "people": [],
  "events": [],
  "summary": "",
  "tags": []
}
```

------------------------------------------------------------------------

## 四、朋友卡片（People）

### 列表卡片结构

-   头像
-   姓名
-   最近互动时间
-   标签

### 详情页结构

-   基本信息
    -   姓名
    -   备注名
    -   标签
    -   重要节日
    -   近况备注
-   时间轴事件列表
    -   日期
    -   事件摘要
    -   标签
    -   图片

### 细节

-   支持手动创建朋友
-   标签体系规则【预设，也支持自由】

------------------------------------------------------------------------

## 五、日历视图（Calendar）

### 展示规则

-   有记录日期，底部会有低饱和度的彩色无边框圆圈
-   不同颜色代表不同事件标签。例如：
    -   蓝色：有知识收获
    -   粉色：陪伴
    -   金色：复合价值


- 支持月/周/日切换

-   点击日期展示当天记录过的朋友头像，头像框颜色对应相应事件的标签

------------------------------------------------------------------------

## 六、祝福 ToDo

### 列表结构
每一行：
-   姓名
-   节日名称
-   生成祝福按钮
-   生成贺卡按钮
-   标记完成按钮
### 互动描述

- 标记完成后，该行颜色变灰；
- 每次点开页面刷新，按日期由近到远排列，已完成的打上标记，下次点开页面自动隐藏。支持展开显示。
- 点击生成祝福按钮，弹出文本框卡片，左上角标AI模型名称，右上角刷新，右下角确认。
- 点击生成贺卡按钮，弹出贺卡[暂时用图片代替]

------------------------------------------------------------------------

## 七、Profile

### 设置
- 支持数据一键导出
- AI接口传入
### 月度总结
- 本月交流最多的小伙伴排行榜
### 标签管理
- 人的标签：工作，研究生，本科生，聚焦在职业和爱好
- 事件的标签：重要程度评级，思考型，陪伴型
